本系列教程向您展示如何开发出可供残障人士访问的网页内容，并为所有人提供更好的用户体验。

这些教程旨在供各种不同角色的人员使用，包括：

- **Web 开发人员**：将能找到许多常见编码挑战的指南和样板解决方案。
- **Web 设计师**：将学习如何创建具有内置包容性设计的网页组件。
- **Web 培训人员**：将找到用于教授人们关于可访问 Web 设计与开发的示例。
- **内容创作者**：将学习以可访问的方式准备其内容的理念和技术。
- **项目经理**：将了解如何将可访问性整合到他们的项目中。

![由 CSS3 提供支持的 HTML5 / 样式、图形、3D 与特效、多媒体和语义化](https://www.w3.org/WAI/content-images/tutorials/html5-badge-h-css3-graphics-multimedia-semantics.png "由 CSS3 提供支持的 HTML5 / 样式、图形、3D 与特效、多媒体和语义化") 这些教程涵盖了基于 Web 项目中常见任务的各种可访问性主题。例如，它们向您展示了如何使用各种 Web 技术（包括 HTML4、HTML5、CSS3、WAI-ARIA、MathML 和 SVG）来提供可访问的图像和表格。教程中解释的概念和技术也同样适用于其他格式。

Web 创作工具（内容管理系统 (CMS)、所见即所得 (WYSIWYG) 编辑器、文字处理器等）通常可以帮助您创建本教程中所示的可访问内容。

## 与 WCAG 的关系

这些教程提供了在不同情况下实现可访问性的最佳实践指南。它们结合了来自不同符合性级别的 WCAG 2 成功准则和技术。每个教程页面都列出了该页面所使用的具体成功准则和技术。关于规范性 [WCAG 2.0](https://www.w3.org/TR/WCAG20/) 和 [WCAG 2.1](https://www.w3.org/TR/WCAG21/) 标准及其支持性材料的更全面介绍，请参阅**[如何满足 WCAG 2：可定制的快速参考](https://www.w3.org/WAI/WCAG21/quickref/)**。

## 其他资源

- **[残障人士如何使用网络](https://www.w3.org/WAI/people-use-web/)：** 介绍了各种残障类型，包括与年龄相关的障碍。
- **[可访问性原则](https://www.w3.org/WAI/fundamentals/accessibility-principles/)：** 介绍了网站、Web 应用程序、浏览器和其他工具的一些 Web 可访问性要求。
- **[WCAG 2 概述](https://www.w3.org/WAI/standards-guidelines/wcag/)：** 介绍了定义如何使网页内容更容易被残障人士访问的国际标准。
- **[创作工具可访问性指南 (ATAG) 概述](https://www.w3.org/WAI/standards-guidelines/atag/)：** 介绍了定义如何让 Web 创作工具支持残障人士可访问性的国际标准。
