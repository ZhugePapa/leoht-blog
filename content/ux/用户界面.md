## 基本概念

你是否曾纳闷，为什么有些应用程序导航起来轻而易举，而另一些却让你感到沮丧和困惑？答案就在于用户界面设计。

想象一下，你正用手机查收邮件，在 Gmail 应用程序中滚动浏览，并毫不费力地删除所有挤满收件箱的垃圾邮件。得益于谷歌体贴入微的界面设计，这款应用的一切设计都非常合理。你可以在不同的收件箱标签页之间切换，通过滑动邮件将其归档，以及批量选择邮件进行高效整理。这些功能感觉非常直观。Gmail 应用程序的用户界面表现得完全符合我们的预期，这确保了我们的体验顺畅且令人愉悦。

用户界面（UI）是人类与计算机交互的空间。它是机器、计算机或软件中处理输入和输出的部分（与进行所有计算的后端相对）。

例如，笔记本电脑的 UI 包括键盘、屏幕和触控板。这些物理输入和输出设备允许你通过输入文本、点击项目以及查看当前状态来使用计算机。笔记本电脑的 UI 还包括屏幕上的图形元素，如菜单和图标，它们允许你使用计算机的操作系统。本质上，UI 让我们能够有效地控制和操作软件或计算机化设备。

UI 设计的目标是让人们尽可能容易地理解和使用计算机。UI 设计越好，产生所需输出所需的输入就越少。想象一下，如果不是通过快速点击删除图标来清除收件箱中的邮件，而是必须在多个设置和菜单项中导航才能找到删除选项。那么为了获得相同的输出，就需要多得多的输入。通过在设计程序时考虑用户的需求、期望和行为，设计师可以创建出为我们节省时间并让生活更轻松的 UI。

## 核心术语

**人机界面 (HMI):** 作为人与机器之间交互点的界面，通常用于工业应用中。HMI 可以是触摸屏、控制面板、按钮、键盘或移动设备，操作员使用它们来控制机械。<sup>1</sup>

**图形用户界面 (GUI):** 标准的计算机 UI，使用户能够通过图标、菜单和其他图形与计算机进行通信。<sup>2</sup> 用户可以使用键盘 and 鼠标，或者在触摸屏上用手指来操作 GUI。无论你是在电脑上操作桌面、访问网站，还是在应用程序中滚动浏览，你都在通过 GUI 进行交互。

**命令行界面 (CLI):** 一种基于文本的 UI，允许人们通过输入文本行（称为命令行）使用键盘与计算机进行交互。<sup>3</sup> 与 GUI 相比，CLI 在执行系统管理任务时效率更高，且占用的网络资源更少，这在与远程服务器协同工作时非常有用。这种技术性界面最常被软件开发人员和系统管理员使用。

**用户体验 (UX):** 用户与产品（如网站、应用程序、系统或服务）交互的整体体验。UI 与 UX 密不可分。设计师依靠[用户体验研究（UX research）](https://thedecisionlab.com/reference-guide/design/ux-research)来做出 UI 设计决策，而优秀的 UI 设计有助于提供更好的 UX。

## 发展历史

在计算机发展的早期，UI 非常简陋。20 世纪 40 年代到 60 年代的计算能力极其有限，由于软件设计师必须优先考虑处理器的使用率，因此几乎没有资源可以分配给 UI 设计。

令人惊讶的是，当时的人们使用打孔的卡片或纸带来与这些计算机进行交互。他们将这些介质输入到计算机中，计算机会输出打印在纸上的文本行 —— 这一过程通常需要耗费数天时间。

直到 1969 年命令行界面（CLI）出现，软件才变得更具交互性。使用 CLI，人们可以通过直接向计算机输入文本来提出请求，并在几小时内（甚至几秒钟内）通过纸张打印件收到响应。

得益于 20 世纪 70 年代视频显示器的采用，UI 从文本形式转变为视觉形式。在基本的用户与计算机交互中，这些早期的计算机屏幕告别了对纸张和墨水的需求。计算机不再将信息打印在纸上供用户阅读，而是直接将输出显示在屏幕上。施乐帕克研究中心（Xerox PARC）开发了最早的基于图形界面的个人电脑之一，它配备了视频显示器、键盘和鼠标。史蒂夫·乔布斯（Steve Jobs）于 1979 年访问了施乐帕克以了解他们的 GUI，并最终利用这些新兴技术开发了苹果的 Macintosh 电脑。<sup>4</sup>

此后的 1985 年，微软创建了 Windows，这是 IBM 第一台个人电脑的操作系统。正是通过这个操作系统，微软将 GUI 推向了大宗市场。苹果和微软都确立了当今图形界面沿用的标准：屏幕顶部的菜单、底部的状态栏以及用于加快输入速度的快捷键。

## 关键人物

**道格拉斯·恩格尔巴特 (Douglas Engelbart):** 在 20 世纪 60 年代，恩格尔巴特发明了各种用于输入和显示数据的设备。恩格尔巴特与斯坦福研究所的一位同事一起，完善了包括摇杆、轨迹球和电脑鼠标在内的多种输入设备。他的工作促成了基本 GUI 的开发，使任何人都可以使用计算机。<sup>5</sup>

**杰夫·拉斯金 (Jef Raskin):** 拉斯金是人机界面领域的专家，在苹果公司成立初期工作。<sup>6</sup> 他于 1979 年启动了 Macintosh 项目，提出了只要个人电脑简单易用，就可以成为普通消费电器的想法。他认为早期的苹果电脑过于复杂，设想了一种低成本且极其用户友好的电脑。

**雅各布·尼尔森 (Jakob Nielsen):** 网页可用性顾问、人机交互专家，被誉为“可用性之王”。他发明了指导 UI 设计的若干可用性原则，称为“尼尔森十大可用性启发式原则”（Nielsen's 10 Usability Heuristics）。<sup>7</sup> 这些启发式原则是广泛的设计规则，旨在通过确保 UI 对最终用户而言直观、一致、高效和灵活来提升用户体验。

## 产生的影响

**提高数字素养**

随着设计师创建出更直观的 UI，普通消费者使用技术成为可能。人们不再需要技术培训就能使用计算机或软件。自那时起，UI 已经成为我们与计算机 and 软件交互不可或缺的一部分 —— 我们每天都在利用现代 UI 的力量！从发送电子邮件这样简单的事情，到视频编辑或建立电子商务商店等更复杂的任务，用户友好界面的发展显著提高了人们的数字素养，使任何人都能使用数字技术执行几乎所有任务。

**以用户为中心的设计**

如今，UI 设计深深根植于真实用户的需求中。当设计师创建或更新 UI 时，他们会把人类行为放在首位，依靠用户研究来决定如何改进数字产品，使其无缝地融入我们现有的生活。设计师试图创建出人们在几乎没有经过培训、新手引导或指导的情况下，就能直接上手的 UI。

UI 设计甚至可以影响我们的行为，鼓励我们点击某个按钮或在注册表单中输入我们的联系信息。用户行为心理学在设计决策中起着重要作用，特别是当目标是提高参与度或优化转化率时。

**工业应用**

UI 设计的进步也带动了其他行业的创新，包括制造业、医疗保健、汽车、酒店、农业和能源领域。例如，人机界面（HMI）使复杂的机械更容易管理，减少了制造业中的错误并提高了效率。<sup>8</sup> 在医疗保健行业，HMI 为专业人员操作复杂的核磁共振成像（MRI）机和 CT 扫描仪提供了一种用户友好的方法。HMI 甚至可以让农民获取有关农业设备的关键信息，并实现对农业操作更精确的控制。

## 争议

**暗黑模式 (Dark Patterns)**

UI 领域的一个主要批评涉及使用设计来影响人类行为的伦理问题。例如，暗黑模式是指欺骗性的 UI 设计，诱骗用户做出购买订阅/商品或提供个人信息等行为。<sup>9</sup> 这些陷阱可能包括使用误导性语言迷惑用户、隐藏额外费用、自动将商品添加到购物车，以及利用内疚或羞耻感来强迫用户做出某种行为（例如订阅邮件列表或进行捐赠）。

以用户为中心的设计*应该*优先考虑用户的目标，但暗黑模式优先考虑的是设计师或企业的利益。这损害了用户体验，并引发了若干伦理问题。

**隐私与数据收集**

批评者还对 UI 设计中用户数据的收集和使用提出了担忧。<sup>10</sup> 设计师经常利用用户数据来做出设计决策并创建个性化体验，这引发了隐私和安全方面的问题。例如，进行用户研究有时意味着在用户不知情的情况下“窥探”其行为，或让其参与 A/B 测试。

设计师必须谨慎使用数据，最好在收集用户信息之前征得他们的同意。收集和分析数据以做出 UI 决策（or 创建动态、个性化的 UI）的设计师也有责任保护这些数据的安全。总的来说，透明度和安全性是确保负责任地使用用户数据的关键。

**无障碍设计的挑战**

虽然数字产品正变得越来越包容，但创建完全无障碍的 UI 依然是一个挑战。设计无障碍界面意味着要在美学与普及包容性之间取得平衡。遗憾的是，许多设计师在为软件、网站和应用程序创建界面时忽略了无障碍设计的重要性，无意中阻碍了残障人士使用该界面。例如，对比度差导致视力障碍人士难以阅读文本，这是数字设计中一个非常普遍的问题。<sup>11</sup>

## 相关 TDL Content

[**认知识别而非回忆 (Recognition Rather Than Recall)**](https://thedecisionlab.com/reference-guide/design/recognition-rather-than-recall)

作为尼尔森十大可用性启发式原则之一，认知识别而非回忆解释了为什么用户更容易识别某些内容，而不是凭记忆回想起某些内容。创建直观的 UI —— 具有可见且易于检索的信息 —— 可以通过限制用户必须记住的信息量，来减少用户所需的认知努力。

[**美学与极简主义设计 (Aesthetic and Minimalist Design)**](https://thedecisionlab.com/reference-guide/design/aesthetic-and-minimalist-design)

我们往往觉得极简主义设计在美学上更令人愉悦。不仅如此，限制页面上的信息量还可以减轻用户的认知负担，让他们更容易完成简单的任务。过多的元素和动态部件会争夺用户的注意力。因此，将 UI 设计重点放在核心要素上，仅向用户提供他们真正需要的信息，是一个非常好的做法。
