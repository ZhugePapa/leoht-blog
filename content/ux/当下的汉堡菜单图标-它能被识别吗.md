摘要：与 10 年前相比，汉堡菜单在今天已成为一个更令人熟悉的设计模式，但针对隐藏式导航的那些经典最佳实践依然适用。

在过去十年中，数字设计领域很少有哪个图标能像“汉堡包”这样引发如此多的争论。[汉堡菜单（hamburger menu）](https://www.nngroup.com/articles/mobile-navigation-patterns/) 因其抽象、堆叠、形似汉堡的外观而得名。最初，它被视为在小屏幕上节省空间的巧妙方法——通过将主导航隐藏在一个按钮后面来实现。虽然这种方法使界面变得整洁，但它也降低了可用性：无论在移动端还是桌面端，用户都容易忽略它，并且需要花费更长的时间来完成任务。

尽管当时这是一种新颖的模式，但在今天，大多数用户已经能够识别汉堡菜单并明白它的含义。然而，许多相同的设计风险依然存在。本文将重新审视我们早期关于汉堡菜单的研究结果，分享关于当下汉堡菜单可识别性的研究，并为汉堡图标的视觉设计提供最佳实践建议。

早在 2015–2016 年，我们进行了一系列[定性研究](https://www.nngroup.com/articles/killing-global-navigation-one-trend-avoid/)和[定量研究](https://www.nngroup.com/articles/hamburger-menus/)，将隐藏式导航模式（如汉堡菜单）与可见的、常驻的导航进行了对比。结果非常明确且一致：

- 用户与隐藏在汉堡菜单图标后面的内容的互动较少。
- 任务成功率下降。
- 任务操作时间增加。
- 整体满意度下降。

这些问题在移动端和桌面端界面上均存在（尽管在桌面端尤为严重）。

我们当时给出的指导建议是，尽可能使导航保持可见，并谨慎使用汉堡菜单——仅在空间受限且确实需要时使用（并且绝不要在桌面端使用）。此外，如果使用汉堡菜单，我们建议遵循几项最佳实践，这些实践在今天依然适用（并在本文末尾进行了总结）。

在过去十年中，汉堡菜单——就像它的同名食物一样——已经成为经典。随着移动优先（mobile-first）设计的普及，它提供了一种整洁、节省空间的解决方案，而当苹果和亚马逊等设计巨头也采用它时，其他设计者纷纷效仿。它的日益普及有助于使其含义标准化：**通过反复接触，用户学会了以更高的信心来识别和解释该图标。**

在为《[Digital Icons That Work: A Comprehensive Guide to Enhancing Experience with Effective Iconography](https://www.amazon.com/dp/B0DF68F4FJ)》一书开展的最新研究中，研究人员向参与者展示了上下文环境中的数十个汉堡图标，并要求他们预测其功能。

虽然这项研究没有像我们以前的研究那样测量任务时间或成功率，但它评估了可识别性：当用户看到该图标时，他们是否将其识别为隐藏菜单？

答案是肯定的。在大多数情况下，参与者能够正确地将汉堡图标识别为主菜单或网站的顶级分类，特别是当它出现在预期的位置并采用标准样式（等长的 3 条堆叠横线）时。

例如，Dick’s Sporting Goods 的移动端网站在标准位置（屏幕的左上角）使用了一个典型样式的汉堡图标来容纳顶级导航。用户在理解这个图标时没有任何问题，即使我们在向参与者展示的截图中去掉了 *Menu*（菜单）标签。

Dickssportinggoods.com（移动端）：参与者将左上角典型样式的汉堡图标理解为菜单，即使截图中去掉了 Menu 标签也是如此。

甚至来自电子商务移动端网站 Vivaia 的双线版汉堡图标，在放置在预期位置时也能被正确理解。

![Vivaia UI 截图，其中的菜单由两条横线表示。](https://lh7-rt.googleusercontent.com/docsz/AD_4nXdH8peGZTe4kVECGGKpeM9h7QVNdg-RV3XAPjvaPaR_rOMyVcoDxWr_2n1YeC7Dv_DxoFeroCxO2h-eaczF_k_vSnznpBUy9QikwBPtO3qN78Fiilzt2dxpIr3hz6KaGLRjEHHqFGK0RA1tk1sY4Mc?key=FKtcoH5A_oCZmPupyLgOsg)

Vivaia.com（移动端）：用户确信双线版的汉堡图标代表网站的主菜单。

对于汉堡图标来说，位置对理解有着重大影响。只要它出现在预期的位置（屏幕左上角），用户通常就会理解该图标的意思是“这里隐藏了分类”。

在我们的研究中，只要保持了典型位置，即使汉堡图标有微小的设计变化，也不会引起重大问题。

例如，Toms 移动端网站在左上角使用了一个类似文档版本的汉堡图标——三条水平线包裹在一个矩形边框内。

大多数用户仍然正确地将其理解为主菜单，尽管该边框带来了一些轻微的困惑。一名参与者将其误认为是文档，而另一名参与者起初有些不确定，随后分析道：

> “通常，那应该是一个菜单项，但它在一个方框里让我觉得它可能是别的什么。不过既然我在其他地方没看到菜单，我就觉得它就是菜单。”

![Toms UI 截图，其中的菜单由方框内的三条横线表示。](https://lh7-rt.googleusercontent.com/docsz/AD_4nXfKtXbjExlXuNqxuCHY8UkBPkqhP0ag_iJCvtIUNagWOhcVX3WUXdwWd21wqv1a3-mlGcYWhAth6vjxuWvc10GEgQKrfyRhPE5dZZHEGZuEBjjSnIdHpGj7QOze9vTRdAWHqfMZ1Ygdd2aKgGYst-4?key=FKtcoH5A_oCZmPupyLgOsg)

Toms.com（移动端）：尽管其放置在左上角，一名参与者仍将带边框包裹的汉堡图标版本误认为是一个文档。

《新闻周刊》（Newsweek）移动端网站使用了一个叠加了箭头符号的汉堡图标（这可能是为了暗示这是一个滑动面板）。

然而，用户忽略了这个额外添加的细节，仍然正确预测出该图标将展示网站的整体分类。同样，图标在左上角的熟悉位置很可能有助于其被正确理解。

![Newsweek UI，其中的菜单由带有向右箭头的三条横线表示。](https://lh7-rt.googleusercontent.com/docsz/AD_4nXcaFVY6zTxj8QIB95bOfHf7zAyM6A1rO3ZVYydClWOyEwQq_0BwxpabfcWnRSwDKYwcuM1MFZhCr7echIHkRRNk8KS2tUAIsl7839NOlqFGv8JyvtDauE5LTEZrvu4XAArP4E0Y4oPMuClHTn6GO2k?key=FKtcoH5A_oCZmPupyLgOsg)

Newsweek.com（移动端）：用户将左上角叠加了箭头的等长三线汉堡图标理解为隐藏菜单。

尽管如此，这些修饰是多余的：改变一个熟悉的形状会带来疑虑，即使它没有完全阻碍用户识别。

汉堡图标在面对轻微的设计变动时仍能保持稳定的可识别性，反映了我们在数字交互中常见的一种现象：曾经不熟悉的模式通过反复接触会演变成潜意识的习惯。然而，**这并不意味着汉堡菜单是万无一失或被所有人理解的**——特别是对于不太懂技术的受众或在不熟悉的布局中。

此外，由于该研究仅考察了可识别性，我们无法评估多年来涉及汉堡菜单任务的任务时间或成功率是否发生了变化。（过去的可用性问题不仅与图标的识别有关，还与可发现性、可记忆性以及对该模式的整体熟悉度有关。）

## 警告：当熟悉度演变成视觉干扰

汉堡菜单已经变得如此普及，以至于它的视觉形式（三条堆叠的横线）已经深深扎根于用户的[心智模型（mental models）](https://www.nngroup.com/articles/mental-models/)中。事实上，以至于**使用类似结构的图标很容易被误认为是汉堡菜单**，即使它们起着完全不同的作用。

当基于横线的图标（例如列表视图或筛选图标）放置在屏幕的左上角（一个与网站主导航紧密相关的区域）时，这种视觉干扰尤其常见。

例如，在苹果备忘录（Apple Notes）的桌面端应用程序中，列表视图图标出现在左上角，紧邻画廊视图图标。尽管该图标包含点状符号（bullet-like dots）以区别于传统的汉堡图标，用户仍然频繁地将其理解为导航菜单。

一位参与者解释道：

> “最左边那个就是汉堡【图标】。这大概是一个可展开的导航。它会展开并展示一些其他的菜单项或功能列表。”

![苹果备忘录 UI 截图，其中的列表视图图标为带有点的三条横线。](https://lh7-rt.googleusercontent.com/docsz/AD_4nXeHlb7oPqVXZ2B8WVZj-71b9T8brDrI_tlIN8rRKccV0uqyhqElmflFoKnS0H1GCn9hZQVU0y7zMSxF6zrcNZPk9Wc7MZtFXlhD_NXI2QAfS9qz1AG2JRTuwLQAKFSf7t1S2ZC_16SMI3k3v4N8Vl8?key=FKtcoH5A_oCZmPupyLgOsg)

苹果备忘录（桌面端）：许多用户将左上角的列表视图图标解释为代表隐藏菜单的汉堡图标。

在另一个例子中，YouTube 移动端应用中由横线组成的筛选图标也引起了混淆。许多用户将其理解为分类或收藏夹列表（这些词汇与导航或列表视图的关系比与筛选的关系更为密切）。

![YouTube UI 截图。其中一项显示了带有垂直短划线的 3 条线，并带有“Search filters”标签。](https://lh7-rt.googleusercontent.com/docsz/AD_4nXdofRm9CIC0zx31Xf03Gi5WqZyaID78PL_cwE9zQujUd4J6sxXDEoo60RmzRcKpe1U_g-sIBJVhLdNvnNOhffEvekCsxQV1PIbUM4JpgfUz9SPLX-SypAHl3FPT7iC1U1oYENbBlP_aMmtzw_95Jnw?key=FKtcoH5A_oCZmPupyLgOsg)

Youtube.com（移动端）：许多用户将屏幕底部上下文菜单中的“搜索筛选（Search filters）”图标理解为分类或收藏列表。（注意，向参与者展示的截图中去掉了这一标签。）

这个例子展示了用户将成组的横线与导航联系在一起的程度是多么深，即使该图标没有放置在左上角。

设计的启示是在用户通常与导航关联的位置使用基于横线的图标（如列表、筛选或内容视图图标）时要格外小心。尤其是在放置在左上角时，这些图标极易被误认为是汉堡菜单。此外，确保为图标添加标签以帮助用户识别和理解。

最近的研究证实，汉堡菜单已被广泛识别：大多数用户将该图标理解为代表隐藏菜单或分类列表。在进行深思熟虑的实现时，它是一个有效的导航工具。

提高汉堡菜单可用性和可发现性的“秘诀”并未改变。为了在今天及未来最大化可用性，请遵循以下最佳实践：

- **使用标准的 3 线图标**：最原始的 3 线汉堡图标在用户中引起了强烈的共鸣。无需刻意创新：坚持使用用户期望的熟悉模式即可。
- **放置在左上角**：用户总是习惯在屏幕左上角寻找隐藏的导航。将菜单放置在其他位置会增加被忽视或误解的风险。
- **避免额外的样式或边框**：装饰性元素（如轮廓或方框）可能会导致用户将该图标误认为其他东西（如文档图标）。
- **为图标添加标签** ***Menu（菜单）***：标签对于经验较少的用户或在不熟悉的语境中引入此模式时尤其有帮助。
- **使用微妙的动画或过渡效果**：如果点击图标会展开导航面板，微妙的动态提示可以增强交互效果并引导用户的预期。
- **避免与其他 3 线图标混淆**：要特别注意区分其他基于横线的图标（如列表视图或筛选），并避免在屏幕和面板的左上角使用类似的图标。
- **适当缩放汉堡图标的大小**：为智能手机设计的汉堡图标在桌面大屏幕上可能会变得近乎隐形。
- **使其看起来可点击**：极端的扁平化设计可能会削弱可发现性并限制用户的点击意愿。
- **在图标周围添加视觉间距（Padding）**：将图标挤在头部或其他 UI 元素附近会降低其可见度。
- **确保足够的对比度**：低对比度的图标会融入背景中，尤其是对于有视觉障碍的用户。
- **提供其他辅助导航方式**：通过行内链接、相关链接列表、显眼的搜索功能以及页脚链接来[辅助汉堡菜单（Support the hamburger）](https://www.nngroup.com/articles/support-mobile-navigation/)。

虽然这些基本的设计原则没有改变，但用户对汉堡菜单的期望和行为确实发生了变化。

另外请记住，无论如何，旧的指导原则依然适用：**如果你不需要隐藏导航，就不要隐藏。** 即使现在大多数用户都认识汉堡菜单，隐藏导航仍然会增加交互成本，因为用户必须多走一步才能访问菜单项。（同时请记住，可识别性的提高并不一定意味着用户在汉堡菜单任务中的操作时间或成功率发生了改变。）

## 结论：最新的判定

汉堡菜单现在已经成为一个大家熟知、可被识别的模式。

但仅凭熟悉度并不能消除其固有的权衡（tradeoffs）。隐藏导航总是会引入[交互成本（interaction cost）](https://www.nngroup.com/articles/interaction-cost-definition/)，因为用户必须额外操作一步才能访问关键选项，且重要的内容仍处于隐藏状态。

当在空间有限的屏幕上经过深思熟虑地使用时，汉堡菜单可以发挥很好的效果。熟悉的设计、一致的放置位置以及对最佳实践的遵循依然至关重要。

**最终判定：汉堡菜单是可行的，但并非万能。** 明智地使用它们，遵循最佳实践，考虑其权衡，并且始终[与用户一起测试它（及其他图标）](https://www.nngroup.com/articles/how-to-test-digital-icons/)，以确保它真正满足了用户的需求。
