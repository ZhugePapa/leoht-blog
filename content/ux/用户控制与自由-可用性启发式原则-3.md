摘要：用户经常会犯错或改变主意。允许他们退出流程或撤销上一步操作，并返回到系统的上一个状态。

Jakob Nielsen 针对用户界面设计的第三条[可用性启发式原则](https://www.nngroup.com/articles/ten-usability-heuristics/)是[用户控制与自由](https://www.nngroup.com/videos/usability-heuristic-user-control-freedom/)。该原则指出：

> *用户在选择系统功能时经常会出错，他们需要一个标记清晰的“紧急出口”来离开不想要的系统状态，而无需通过繁琐的对话。支持撤销（undo）和重做（redo）。*

优秀用户体验的一部分在于培养用户对他们正在使用的用户界面（UI）的控制感。**用户应该能够快速纠正错误或撤回已做出的选择**。轻松摆脱困境的能力鼓励了用户的探索，这有助于对功能的学习和发现。它还能增加整体的使用量和销售额（在探索产品空间的情况下）。相反，当 UI 不支持这些操作时，用户会感到被困，并通常会反馈不满意。

有几种 UI 控件通常允许人们**返回到系统的上一个状态**：

- “返回（Back）”链接，用于将用户送回上一个页面或屏幕
- “取消（Cancel）”链接，允许用户退出任务或多步骤流程
- “关闭（Close）”链接，允许用户关闭新视图
- “撤销（Undo）”选项（以及相应的重做（redo）选项），允许用户撤销对 UI 元素所做的更改

## 始终允许用户后退一步

每当用户点击链接打开新页面、屏幕或视图时，他们应该始终能够返回到他们来的地方。在 Web 上，用户依靠浏览器的“返回”按钮导航到上一个页面。在可用性测试中，我们经常看到一些用户在浏览网站时默认使用浏览器的“返回”按钮，而不是使用网站自身的导航（如面包屑）。这也是我们建议[不要盲目在新建标签页中打开链接](https://www.nngroup.com/articles/new-browser-windows-and-tabs/)的原因之一，因为有些用户没有注意到他们处于新标签页中，从而在想要返回时感到困难。

**绝不要通过禁用浏览器“返回”按钮来阻止用户离开你的网站。** 一些网站故意使其网站具有粘性，阻止用户返回搜索引擎。虽然如果你使用这种策略，用户在你的网站上停留的时间会多几秒钟（因为他们无法离开），但你很快就会激怒他们——更别想指望他们与你做生意了！这种设计选择往往是优先考虑[虚荣指标（vanity metrics）](https://www.nngroup.com/articles/vanity-metrics/)而不是实际 UX 目标的结果。网站应该通过提供有价值的内容来吸引用户留下来，而不是通过禁用“返回”按钮来困住用户。

在一些在线表单中也可以发现无法正常工作的浏览器“返回”按钮，因为向后移动可能会破坏底层的逻辑。在某些情况下，用户可以使用浏览器“返回”按钮，但没有返回到上一个屏幕，而是收到了超时消息并丢失了他们的工作。在可能的情况下，**构建用户 *可以使用* 浏览器“返回”按钮的表单**。如果设计使得这变得不可能，请在用户点击浏览器的“返回”按钮时向他们发出警告，并为他们提供取消此操作的机会。设计师还应该探索创建“返回”链接或可点击的进度条，以帮助用户退回表单而不会丢失工作。

## 使用返回链接时满足用户的预期

当面对[弹窗或灯箱（overlay or lightbox）](https://www.nngroup.com/articles/overuse-of-overlays/)时，用户经常会感到迷失方向，认为自己处于一个新页面中——特别是在弹窗占据了整个屏幕的情况下。因此，为了返回到上一个屏幕，用户经常使用浏览器的“返回”按钮，而不是点击“关闭”链接或“X”图标。这种操作的效果是将用户带回两步，而不是一步，从而造成困惑和迷失方向。

在最近的一次移动端可用性测试课中，一名参与者在全屏弹窗上使用了浏览器的“返回”按钮，结果被带出了弹窗，*并且*也带出了他想要返回的页面。该参与者评论说：“我不希望被带离那个页面……如果我点击返回，它会把我带出我所在的页面。它没有把我带回我弹出前的页面。”

当用户有可能将弹窗误认为是新屏幕时——无论是在桌面端还是移动端——**请确保“返回”按钮具有与“关闭”链接相同的效果**。

![一个 GIF 显示了服装零售商 Asos.com。当用户从产品列表页面中选择一双鞋子，并从该页面中选择一个链接以查看配送信息时，会出现一个全屏弹窗。使用浏览器的返回按钮（而不是关闭图标）会将用户带回产品列表，而不是返回到用户选择的那双鞋子的产品概览页面。](https://media.nngroup.com/media/editor/2020/11/20/asos_back.gif)

在 ASOS.com 上，当用户选择“免费配送与退货”链接时，会出现一个全屏弹窗。点击浏览器的“返回”按钮会将用户返回到产品列表，而不是产品概览页面。

![展示了 Wayfair.com 的两个屏幕截图。屏幕 1 显示了一张扶手椅的产品概览页面。在产品标题下方是平均评分（4.5）和阅读评论的链接。点击该链接会打开一个占据全屏的抽屉式弹窗。屏幕顶部出现一个返回链接，用于将用户带回点击评论链接之前正在查看的屏幕。](https://media.nngroup.com/media/editor/2020/11/20/wayfair_compressed.png)

在 Wayfair.com 上，点击“评论”链接会打开一个扩展至全屏的抽屉。两个返回按钮（网站自身的和浏览器的）都将用户带到初始的产品概览页面，这正是用户所期望的。用户也可以从左向右滑动来关闭抽屉。

在移动应用中，用户倾向于将他们在 Web 体验中积累的关于“返回”按钮的知识迁移过来，并对其抱有相同的期望：即这是一种后退一步的方法（无论用户是从哪个方向来的）。不幸的是，设计师往往将其定义为在应用的智能架构（IA）中向上移动的一种方式。结果，用户很容易迷失方向。在下面的例子中，用户可以从右向左滑动以在 BBC 新闻的应用中阅读另一篇新闻故事（其顺序对应于它们在新闻列表页面上垂直排列的方式）。然而，用户无法使用“返回”按钮返回到他们通过滑动访问的故事；使用“返回”按钮会将用户在智能架构中向上一级带回到文章列表页面，而不是返回到他们之前访问过的文章。

![BBC 新闻应用的 GIF 显示了用户点击主页上的一个故事，然后从右向左滑动以移动通过故事。当用户选择应用中的返回按钮时，他们被带回到了列出所有故事的主页面。](https://media.nngroup.com/media/editor/2020/11/20/bbc_app_compressed.gif)

在用户点击 BBC 新闻应用上的一篇新闻故事后，他们可以滑动浏览列表中的其他文章。“返回”按钮将用户带回智能架构的上一级（返回列表页面），而不是返回到用户之前访问过的文章。

## 让退出链接易于发现

就像在物理空间中一样，**出口应该易于寻找且标志醒目**，以便在需要时能够被发现。

在放置“关闭”、“退出”或“取消”标志时，遵循[设计标准](https://www.nngroup.com/videos/usability-heuristic-consistency-standards/)，以便用户可以在他们通常期望的地方轻松找到它们。例如，在灯箱弹窗上，用户通常期望在弹窗的右上角看到一个“X”图标。

![Harper's Bazaar 网站上一篇文章的图片。文章上方出现一个模态弹窗。关闭链接出现在模态弹窗的外部，并且由于与背景（即文章的图片）缺乏对比而很容易被忽略。](https://media.nngroup.com/media/editor/2020/11/20/harpers_bazaar_compressed.jpg)

在 Harper’s Bazaar 的网站上，一个弹窗邀请用户考虑订阅。关闭链接出现在弹窗外部的左下角。这种不同寻常的放置位置，加上与背景较差的对比度，使其很难被看到。

除了在预期的位置放置“退出”标志之外，使用[通用图标](https://www.nngroup.com/articles/icon-usability/)以便用户理解该链接的作用。[为图标搭配文本标签](https://www.nngroup.com/articles/icon-usability/)，或者用纯文本代替图标。例如，使用“返回（Back）”代替 `<`，使用“关闭（Close）”代替 `X`。

## 允许用户轻松取消操作

无论用户是开始购买、转账还是下载大文件，她都应该能够在任何时候取消该任务。“取消”选项应该易于寻找且能够快速执行。尽管在多步骤流程中，用户可以使用“返回”链接来代替“取消”，但最好还是提供一个“取消”链接以避免不必要的点击。

![用户开始在 Twitter 上撰写推文的屏幕截图。已点击取消链接，从而生成了一个对话框，询问用户是想删除草稿还是保存它。出现了另一个取消链接，以允许用户撤销该操作。](https://media.nngroup.com/media/editor/2020/11/20/twitter_cancel_compressed.jpg)

当用户开始撰写推文时，他们可以通过选择屏幕左上角的“取消”来轻松取消该任务。这样做时，Twitter 会为用户提供删除消息或保存草稿的选项。这一选择确保了用户不会意外丢失工作。

在移动端，一些设计师试图通过使用“X”图标而不是“取消”来节省空间。不幸不足，有时[“X”可能会被误认为是“关闭”而不是“取消”](https://www.nngroup.com/articles/cancel-vs-close/)。在某些界面中，区分这两者之间的差异对于避免用户出错非常重要。如果“X”表示“关闭”*并且*“取消”，那么请使用明确的文本标签（取消）或向用户提供警告对话框，以避免丢失用户的工作。

![Wealthfront 应用（理财应用）的屏幕截图。用户已经开始了一个多步骤的流程，将资金从投资账户转移到活期账户。为用户提供了确认转账、通过“X”图标取消转账以及通过点击返回箭头图标在转账过程中后退一步的选项。](https://media.nngroup.com/media/editor/2020/11/20/wealthfront_compressed.jpg)

在 Wealthfront 的应用中从投资账户转账时，用户需要在连续的屏幕上遵循几个步骤。在屏幕顶部，用户可以使用箭头后退一步，或者选择“X”来取消转账。然而，目前还不完全清楚按下是否会保存转账参数还是完全取消转账。在这种情况下，它会完全取消转账，因此使用“取消”标签会更好。

## 支持撤销（Undo）

当用户对系统的状态做出更改时，他应该能够轻松地撤销该操作。想象一下，不小心删除了整整一段文字却无法找回！

在界面上可以通过多种方式支持“撤销”，而不仅仅是通过一个简单的“撤销”按钮。例如，如果用户误将一件商品添加到购物车中，他们应该能够通过移除该商品来撤销该更改。

![CVS 药房网站购物车的图片。购物车中添加了一些抗过敏眼药水。商品下方出现一个移除链接，允许用户将商品从购物车中移除。](https://media.nngroup.com/media/editor/2020/11/11/cvs_checkout.png)

在电子商务网站上，例如上面 CVS 的例子，用户可以通过选择“移除（Remove）”来撤销将商品添加到购物车的操作。

![iOS 针对 BBC 新闻应用的通知设置截图。为用户提供了开启或关闭通知和声音的选项，并且还可以选择和取消选择通知的显示方式（例如，在锁屏上、在通知中心或在横幅中）。](https://media.nngroup.com/media/editor/2020/11/20/ios_notification_settings_compressed.jpeg)

在更改通知设置时，用户可以通过使用开关和复选框来撤销更改。

![Mattamy 网站上的互动户型图。用户通过从页面左侧出现的面板中选择添加项，对户型图做出了一些更改。在户型图本身的右侧显示了几个没有标签的图标。一个图标是向后的箭头，在此图片中已被作者标出。](https://media.nngroup.com/media/editor/2020/11/20/mattamy_floorplan_compressed.jpg)

住宅建造商 Mattamy 在其网站上提供互动式户型图，为用户提供了一种探索几种可能的结构变化的方法。为了撤销编辑，用户要在左侧面板中取消勾选结构更改。不幸的是，重置按钮（被标出）很容易被误认为是撤销按钮。

**If 用户可能在短时间内连续做出许多操作，请支持多次“撤销”和“重做”**。仅撤销上一步操作往往是不够的。对于桌面端用户，为专业用户支持[标准的键盘快捷键](https://www.nngroup.com/articles/flexibility-efficiency-heuristic/)也是一个好主意。

## 确保撤销易于发现

还记得 [iOS 中的摇一摇撤销（shake-to-undo）功能](https://www.nngroup.com/articles/ios-7/)吗？这一功能因极难被发现而众所周知地未被充分使用。

对于桌面端设计，不要假设你的用户会知道他们可以使用键盘快捷键。在 UI 上为用户提供一个可见的撤销操作选项是一个好主意。这一功能应该在用户通常期望的地方。在文字处理器和设计软件等所见即所得（WYSIWYG）应用中，“撤销”通常在工具栏或应用的菜单中。在没有[浏览器边框（chrome）](https://www.nngroup.com/articles/browser-and-gui-chrome/)的 Web 应用中，“撤销”选项通常是情境化的——它可能会出现在临时的轻量级提示栏（snackbar）或[右键快捷菜单（contextual menu）](https://www.nngroup.com/articles/contextual-menus/)中。

![Microsoft Word 工具栏的图片。编辑菜单分类已被打开，“撤销键入”菜单项已被选择。菜单项旁边显示了向用户提示的说明，向他们展示可以通过使用键盘快捷键来激活此命令。](https://media.nngroup.com/media/editor/2020/11/20/microsoft_word_toolbar_compressed.jpg)

在 Microsoft Word 中——就像在许多其他所见即所得应用中一样——用户可以从“编辑”菜单中找到“撤销”和“重做”选项。用户还可以看到这些操作的键盘快捷键。

如果“撤销”选项是情境化的，请确保它在 UI 中具有良好的可见性。在谷歌云端硬盘（Google Drive）上，当用户对文件做出更改时，窗口左下角会出现一个轻量级提示栏，以确认更改已做出并提供撤销选项。不幸的是，该消息只出现几秒钟。用户需要知道他们想要撤销并迅速行动才能使用此功能。（也可以使用键盘快捷键，但 UI 中并未列出这些快捷键。）

![谷歌云端硬盘文件夹内容的图片。此文件夹中包含两个文档，下方出现一条通知，告诉用户一个文档已被成功重命名。在此通知中有一个撤销该操作的链接。](https://media.nngroup.com/media/editor/2020/11/11/google_drive.png)

在谷歌云端硬盘中，只有在用户采取行动时才提供“撤销”选项。它在操作完成后以轻量级提示栏的形式出现在屏幕底部，但在几秒钟后就会消失。

## 结论

为了确保用户在使用你的产品时有一种自由和控制感，他们应该能够轻松地放弃任务、后退一步并撤销他们对系统状态所做的更改。通过精心设计清晰的退出点和“撤销”功能，你可以让用户感到对体验拥有控制权，而不是任由你的设计摆布。
