按钮模式

## 关于此模式

[按钮 (button)](https://w3c.github.io/aria/#button) 是一种使用户能够触发操作或事件的微件（widget），例如提交表单、打开对话框、取消操作或执行删除操作。通知用户按钮将启动对话框的常用惯例是在按钮标签后附加“…”（省略号），例如“另存为…”。

除了普通按钮微件外，WAI-ARIA 还支持另外两种类型的按钮：

- 切换按钮（Toggle button）：一种具有双状态的按钮，可以是关闭（未按下）或开启（已按下）。为了向辅助技术表明按钮是一个切换按钮，需要为 [aria-pressed](https://w3c.github.io/aria/#aria-pressed) 属性指定一个值。例如，音频播放器中带有“静音”标签的按钮，可以通过将按下状态设置为 true 来指示声音已静音。**重要提示：** 切换按钮在状态改变时，其标签不能改变，这一点至关重要。在此示例中，当按下状态为 true 时，标签仍保持为“静音”，因此屏幕阅读器会读出类似“静音切换按钮已按下”的内容。相反，如果设计要求按钮标签从“静音”更改为“取消静音”，则不需要使用 aria-pressed 属性。
- 菜单按钮（Menu button）：如 [菜单按钮模式 (menu button pattern)](https://www.w3.org/WAI/ARIA/apg/patterns/menu-button/) 中所述，如果按钮的 [aria-haspopup](https://w3c.github.io/aria/#aria-haspopup) 属性设置为 `menu` 或 `true`，辅助技术就会将其识别为菜单按钮。

### 注意

按钮执行的操作类型与链接的功能截然不同（参见 [链接模式 (link pattern)](https://www.w3.org/WAI/ARIA/apg/patterns/link/)）。微件的外观和角色（role）与其提供的功能相匹配是至关重要的。然而，有些元素偶尔具有链接的视觉样式，但执行的却是按钮的操作。在这种情况下，为该元素赋予 `button` 角色有助于辅助技术用户理解该元素的功能。不过，更好的解决方案是调整视觉设计，使其与功能和 ARIA 角色相匹配。

## WAI-ARIA 角色、状态和属性

- 该按钮的角色为 [button](https://w3c.github.io/aria/#button)。
- 按钮具有一个可访问标签（accessible label）。默认情况下，可访问名称（accessible name）是从按钮元素内部的任何文本内容计算得出的。然而，也可以通过 [aria-labelledby](https://w3c.github.io/aria/#aria-labelledby) 或 [aria-label](https://w3c.github.io/aria/#aria-label) 来提供。
- 如果存在按钮功能的描述，按钮元素会将 [aria-describedby](https://w3c.github.io/aria/#aria-describedby) 设置为包含该描述的元素的 ID。
- 当与按钮关联的操作不可用时，按钮会将 [aria-disabled](https://w3c.github.io/aria/#aria-disabled) 设置为 `true`。
- 如果按钮是切换按钮，它会具有 [aria-pressed](https://w3c.github.io/aria/#aria-pressed) 状态。当按钮被切换为开启时，该状态的值为 `true`，当切换为关闭时，该状态的值为 `false`。
