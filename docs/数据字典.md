# Leoht 数据字典 v1.0

> 本文档定义 Leoht 知识系统中所有数据对象的字段、类型、约束与关系。
> 当前实现状态：Astro 静态站点 + Markdown 内容集合（无数据库）。
> 后端方案见 [后端技术方案.md](./后端技术方案.md)。

---

## 1. 数据对象总览

| 对象 | 说明 | 当前实现 | 目标实现 |
| --- | --- | --- | --- |
| **Card** | 文章 / 卡片——核心内容单元 | Markdown frontmatter（Zod schema） | 数据库表 + API CRUD |
| **Tag** | 标签——扁平横向组织 | 隐式（Card.tags 数组） | 独立表 + 聚合页 + 使用统计 |
| **Link** | 卡片间关联关系 | Card.related（slug 列表） | 独立表 + 关系类型 + 反向查询 |
| **Domain** | 领域——Domain 分组视图 | TypeScript 常量（domains.ts） | 数据库表 + 可后台管理 |
| **User** | 用户——登录 / 管理 | 未实现 | Auth + 角色（owner / reader） |
| **Subscription** | 邮件订阅 | 未实现 | 数据库表 + Resend 发信 |
| **Collection** | 专题合集 | 未实现（Phase 2） | 数据库表 + Card 关联表 |
| **Comment** | 读者评论 | 未实现 | 数据库表 + 审核流程 |

## 2. Card（文章 / 卡片）

> 核心内容单元。一篇文章 = 一张 Card。

### 2.1 字段定义

| 字段 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| `id` | string (uuid) | 系统生成 | — | 主键 |
| `title` | string | ✅ | — | 标题 |
| `slug` | string | 可选 | 取文件名 | URL 友好标识 |
| `summary` | string | — | `""` | 一句话核心观点，用于卡片 / 列表摘要 |
| `content` | text (markdown) | ✅ | — | 正文（Markdown body） |
| `type` | enum | ✅ | — | `article` / `note` / `highlight` / `principle` / `effect` / `case` / `retro` / `reading` / `spark` |
| `status` | enum | ✅ | `inbox` | `inbox` / `draft` / `published` / `archived` |
| `tags` | string[] | — | `[]` | 标签 slug 列表 |
| `domain` | string? | — | `null` | 领域 slug（可选） |
| `source_title` | string? | — | `""` | 来源标题 |
| `source_author` | string? | — | `""` | 来源作者 |
| `source_url` | string (url)? | — | `""` | 来源链接 |
| `published_at` | date? | — | `null` | 发布日期 |
| `related` | string[] | — | `[]` | 关联 Card slug 列表（自动生成 Link） |
| `created_at` | datetime | 系统生成 | — | 创建时间 |
| `updated_at` | datetime | 系统生成 | — | 更新时间 |

### 2.2 type 枚举

| 值 | 中文名 | 说明 |
| --- | --- | --- |
| `article` | 文章 | 完整长文 |
| `note` | 笔记 | 简短记录 |
| `highlight` | 高亮 | 摘录句 |
| `principle` | 原理 | 设计原则 |
| `effect` | 效应 | 心理学效应 |
| `case` | 案例 | 案例分析 |
| `retro` | 复盘 | 项目复盘 |
| `reading` | 读书 | 读书笔记 |
| `spark` | 灵感 | 灵感碎片 |

### 2.3 status 枚举

| 值 | 说明 | 对外可见 |
| --- | --- | --- |
| `inbox` | 收件箱——刚记录，未整理 | ❌ |
| `draft` | 草稿——正在写 | ❌ |
| `published` | 已发布 | ✅ |
| `archived` | 归档——不再展示 | ❌ |

### 2.4 当前 Astro Schema（content.config.ts）

```typescript
z.object({
  title: z.string(),
  slug: z.string().optional(),
  summary: z.string().default(""),
  type: z.enum(["article", "note", "highlight", "principle", "effect", "case", "retro", "reading", "spark"]),
  status: z.enum(["inbox", "draft", "published", "archived"]),
  tags: z.array(z.string()).default([]),
  domain: z.string().optional(),
  source_title: z.string().nullable().optional().default(""),
  source_author: z.string().nullable().optional().default(""),
  source_url: z.union([z.string().url(), z.literal(""), z.null()]).optional().default(""),
  published_at: z.coerce.date().optional(),
  related: z.array(z.string()).default([]),
})
```

## 3. Tag（标签）

> 扁平横向组织机制。当前隐式存在于 Card.tags 数组中，无独立表。

### 3.1 字段定义

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `id` | string (uuid) | 系统生成 | 主键 |
| `name` | string | ✅ | 标签显示名（中文） |
| `slug` | string | ✅ | URL 标识（唯一） |
| `group` | enum | — | `type` / `stage` / `behavior` / `scene`（见 §3.2） |
| `description` | string? | — | 标签页展示用 |
| `usage_count` | int | 系统统计 | 被引用次数（自动计算） |
| `created_at` | datetime | 系统生成 | — |

### 3.2 标签分组

| 分组 | 含义 | 示例 |
| --- | --- | --- |
| `type` | 内容是什么 | 心理学、案例、方法论、框架、原则、数据 |
| `stage` | 设计阶段 | 研究、定义、设计、验证、评估、增长 |
| `behavior` | 行为层级 | 认知、决策、行为、情绪、注意力、动机 |
| `scene` | 应用场景 | 注册登录、Onboarding、订阅转化、AI 对话 |

  ├──< Link >── Card (自引用，source → target)

## 11. 编号体系

Card 有三段编号 `{global}.{domainAbbr}.{domainSeq}`：

| 段 | 含义 | 生成规则 |
| --- | --- | --- |
| `global` | 全局期号（罗马数字） | 按发布时间升序，最早 = i（`toRoman(total - i)`） |
| `domainAbbr` | 领域缩写（3 大写字母） | `DOMAIN_ABBR[domain] ?? domain.slice(0,3).toUpperCase()` |
| `domainSeq` | 域内序号 | 同领域内按时间升序递增 |

示例：`i.LDR.1` = 全刊第 1 篇 / 领域 leadership / leadership 域内第 1 篇。
