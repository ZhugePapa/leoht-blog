## 2. 文章类型（Type）

9 种类型，当前全部为 `article`。

| 类型 | 中文名 | 说明 | 适用模板 | 当前数量 |
| --- | --- | --- | --- | --- |
| `article` | 文章 | 完整长文，自由结构 | 无固定模板，自由写 | 3 |
| `note` | 笔记 | 简短记录 | 单段，不需分节 | 0 |
| `highlight` | 高亮 | 原文摘录 | 原文摘录 + 我的理解（两块） | 0 |
| `principle` | 原理 | 设计原则 | 核心观点 + 应用场景 | 0 |
| `effect` | 效应 | 心理学效应 | 原文摘录 + 我的理解 + 应用场景 | 0 |
| `case` | 案例 | 案例分析 | 背景 + 分析 + 启示 | 0 |
| `retro` | 复盘 | 项目复盘 | 背景 + 过程 + 反思 + 下一步 | 0 |
| `reading` | 读书 | 读书笔记 | 书籍信息 + 摘录 + 我的理解 | 0 |
| `spark` | 灵感 | 灵感碎片 | 一句话，不分节 | 0 |

### 内容模板建议（按 type 分化）

| 类型 | 模板结构 | "我的理解" | 
| --- | --- | --- |
| `article` | 自由写，不套模板 | 融入正文，不单设 | 
| `note` | 单段 | 不需要 | 
| `highlight` | 原文摘录 → 我的理解 | **核心两块**，理解可短至一句 | 
| `principle` | 核心观点 → 应用场景 | 融入核心观点 | 
| `effect` | 原文摘录 → 我的理解 → 应用场景 | **独立一块** | 
| `case` | 背景 → 分析 → 启示 | 融入分析 | 
| `retro` | 背景 → 过程 → 反思 → 下一步 | 融入反思 | 
| `reading` | 书籍信息 → 摘录 → 我的理解 | **独立一块** | 
| `spark` | 一句话 | 不需要 | 

**原则**：摘录型（highlight / effect / reading）保留"原文摘录 + 我的理解"两块结构，理解可以只有一句话；长文型（article / case / retro）自由写；灵感型（note / spark）不分节。

### 3.2 标签分组维度（4 组）

| 分组 | 含义 | 示例 |
| --- | --- | --- |
| `type` | 内容是什么 | 心理学、案例、方法论、框架、原则、数据、视觉 |
| `stage` | 设计阶段 | 研究、定义、设计、验证、评估、增长 |
| `behavior` | 行为层级 | 认知、决策、行为、情绪、注意力、动机、信任、层级、对比、协作、沟通 |
| `scene` | 应用场景 | 注册登录、Onboarding、订阅转化、信息架构、表单设计、空状态、AI 对话、排版、设计评审、交互设计 |

### 3.3 标签规则

- 一篇文章 ≥1 个标签，建议 3-7 个
- 标签优先表达"可复用场景"，不只是文章表面主题
- 标签可点击进入标签聚合页
- 同义标签支持合并（管理后台 P2 功能）
- 标签分组在后台可调（当前分组是建议值，未在代码中实现）
