## 0. 模型速览（v1.1）

```
领域（7 个稳定骨架）：behavior / ux / growth / design-system / thinking / ai-product / org
标签（6 组语义索引）：scenario / problem / product / ui / concept / usage
类型（3 种来源形态）：original / knowledge / quote
```

**核心原则**：`domain + tags` = 语义；`type` = 来源形态，不参与语义分类。

## 2. 类型（Type）

3 种来源形态，不参与语义分类。当前 3 篇全部为 `original`。

| 类型 | 中文名 | 核心能力 | 是否重写原文 | 当前数量 |
| --- | --- | --- | --- | --- |
| `original` | 个人原创 | 创造知识 | ✔️（自己写） | 3 |
| `knowledge` | 知识整理 | 结构化外部知识 | ✔️ 重组 | 0 |
| `quote` | 引用摘录 | 保存原文 | ❌ 严禁改动 | 0 |

### 类型模板建议

| 类型 | 模板结构 | 个人观点 |
| --- | --- | --- |
| `original` | 自由写，可参考 `thinking_type` 分化结构 | ✔️ 强，融入正文 |
| `knowledge` | key_points → structure → summary → interpretation | ⚠️ 弱，只在 interpretation 表达 |
| `quote` | 原文摘录 → annotation（why_important / usage_note） | ❌ 极弱，仅 annotation |

**thinking_type**（仅 `original`）：`framework / opinion / method / analysis / case_reflection`。

### 3.1 当前在用标签（12 个，迁移后归入 6 组）

| 旧标签 | 新 slug | 新分组 | 出现次数 |
| --- | --- | --- | --- |

| 对比 | `contrast` | ui | 1 |
| 交互设计 | `interaction` | ui | 1 |
| 方法论 | `framework` | usage | 1 |

### 3.2 标签分组（6 组）

| 分组 | 含义 | 当前标签 |
| --- | --- | --- |
| `scenario` | 场景——按任务找知识 | ai-conversation / design-review / communication / collaboration |
| `problem` | 问题——设计解决什么 | decision-making / trust |
| `product` | 产品类型——设计环境 | （暂无） |
| `ui` | 设计对象——作用在哪 | typography / visual / hierarchy / contrast / interaction |
| `concept` | 概念——理论基础 | （暂无） |
| `usage` | 内容用途——东西是什么 | framework |

### 3.3 标签治理规则

- 一篇文章 ≥1 个标签，建议 3-7 个
- 标签必须归属某一类（scenario / problem / product / ui / concept / usage）
- 禁止重复语义标签（如 `usability` vs `user-friendly`，二选一）
- `scenario` 优先于 `concept`（能用场景就不用概念）
- 每条内容标签上限：`scenario` ≤ 2 / `problem` ≤ 3 / `ui` ≤ 5 / `concept`/`product`/`usage` 各 ≤ 3
- 标签分组在后台可调（当前分组是迁移建议值，未在代码中实现）
