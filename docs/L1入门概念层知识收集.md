## 3. 推荐采集顺序

如果先做 30 篇第一批文章，建议按这个顺序：

1. `cognitive load UX`：先建立 UX 基础认知。
2. `mental model interface design`：建立用户预期与系统模型。
3. `visual hierarchy usability`：连接视觉设计和可用性。
4. `choice overload UX`：用于首页、内容库、筛选器和标签系统。
5. `default effect product design`：用于产品默认配置与行为引导。
6. `loss aversion UX`：用于定价、试用、收藏和取消流程。
7. `trust interface design`：用于知识系统的可信表达。
8. `emotional design UX`：用于阅读体验与情绪反馈。
9. `motivation theory UX`：用于订阅、收藏、持续阅读和复访。
10. `eyetracking web usability`：用于后续页面布局与文章阅读体验。
