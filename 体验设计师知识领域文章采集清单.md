# 体验设计师知识领域文章采集清单

版本：v0.1  
日期：2026-06-13  
用途：为「体验设计师个人知识系统」收集文章、案例、摘录、论文与设计资料  
排序方式：按知识理解难度从初级过渡到高级

---

## 0. 使用方式

这份清单用于指导前期文章数据采集。采集时不要只保存链接，而要把每篇内容转成可复用知识卡片。

建议每条采集内容至少记录：

```text
标题
原文链接
作者 / 来源
发布时间
所属 Domain
所属 Topic
难度等级
内容类型
核心观点（一句话）
适用场景
关键摘录
我的理解
相关标签
相关内容链接
```

## 1. 难度分层

| 等级  | 名称   | 适合采集的内容             | 判断标准          |
| --- | ---- | ------------------- | ------------- |
| L1  | 入门概念 | 定义、术语、基础原则、常见模型     | 读完能回答“这是什么”   |
| L2  | 基础方法 | 可执行方法、流程、检查清单、模板    | 读完能回答“怎么做”    |
| L3  | 场景应用 | 具体产品场景、案例拆解、模式分析    | 读完能回答“用在哪里”   |
| L4  | 系统策略 | 指标、组织、实验、复杂系统、跨团队协作 | 读完能回答“如何影响结果” |
| L5  | 研究前沿 | 学术论文、新技术、新范式、原创框架   | 读完能回答“未来会怎么变” |

采集建议：

- L1-L2：优先建立稳定知识底座。
- L3：最适合转成公开文章和案例卡片。
- L4：适合沉淀成方法论、专题和设计决策库。
- L5：适合做趋势观察、AI UX、HCI 研究前沿。

## 10. 每篇文章的采集判断标准

采集前问 5 个问题：

1. 这篇内容能解释一个用户行为吗？
2. 这篇内容能指导一个设计决策吗？
3. 这篇内容能补充一个方法或模板吗？
4. 这篇内容能关联到至少 2 个已有标签吗？
5. 这篇内容未来能被复用到文章、案例或项目复盘中吗？

如果 5 个问题中至少 3 个为“是”，值得采集。
